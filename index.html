Design principles
◈
Numbers are not decoration
Color on numbers encodes direction — favorable, unfavorable, at risk. Never applied in GL views where column position already carries that meaning.
◎
IDR-first, always
All amounts default to IDR canonical: Rp 1.250.000,00. Period thousands, comma decimal. Multi-currency shows FCY + kurs + IDR equivalent — never collapses to one column.
⊞
Density over whitespace
ERP users process dozens of records at a time. Tables are dense by design. Spacing is functional — it creates hierarchy, not breathing room.
◇
Tax is always visible
DPP, PPN, and PPh are always shown as separate columns. Tax is never collapsed into a total. PSAK compliance depends on explicitness.
○
State over instruction
Fields communicate their state through visual properties — not helper text. A read-only field looks different from a disabled field. A due-soon row reads orange without a tooltip.
◉
One font per role
Manrope for all UI text. Inconsolata for all numbers, IDs, codes, and monetary values. Bebas Neue for page and section titles only. Never mixed within a single element.
