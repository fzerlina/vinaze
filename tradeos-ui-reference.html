✓ Do
  • One primary button per context — the highest-commitment action
  • Secondary always pairs with a primary — never standalone
  • Text links for navigation and low-weight actions inside prose or below cards
  • AI command buttons always carry the sparkle icon, even when space is tight
  • All button labels in Camel Case — except text links, which use sentence case
  • Disabled state uses the same shape, reduced to 40% opacity
✗ Don't
  • Two primary buttons side by side — creates ambiguity about what to do
  • Secondary button alone without a primary — it has no anchor
  • AI commands without the sparkle icon — users won't know it's AI-powered
  • Text links inside table cells or data rows — use the tbl-link underline pattern instead
  • Mixed label casing within the same button group
  • Uppercase or ALL CAPS labels — they belonged to the previous system

AI command buttons
Always sparkle-first
The sparkle icon is the only visual signal that an action will invoke AI. It must always appear — even if the button label already implies it (e.g. "Summarise"). Never replace it with another icon, never omit it for space.
Sparkle always leadsIcon left of label — never right, never below. The icon is the signal; the label explains the action.
Aa
Sentence case labelsAI command labels use sentence case — "Explain variance", not "Explain Variance". They describe what the AI will do, not a UI action the user performs.
⬜
Ink outline, never filledAI commands use the same ghost treatment as the secondary button but with ink (#1a1a1a) border and text — never the indigo primary fill. This keeps them visually separate from navigation actions.
⏳
Loading state: spinner replaces sparkleWhile the AI is responding, swap the sparkle for a spinner in the same position. Label changes to "Working…". Button becomes disabled.

Table A — General Ledger
Why all GL numbers are plain black: Debit does not mean "bad." A debit to cash increases your balance — it's a positive event. A debit to payroll is normal business operation. The balance column is a running computed figure with no inherent good/bad meaning — blue would add no semantic value there either. Xero and QuickBooks use plain black for all GL values — the column header labels ("Debit" / "Credit") carry the structural distinction, not the numbers themselves. Clickable references (INV-, REC-) use underline rather than blue — so the link affordance doesn't mimic or compete with the number color system.

📐
Consistent base across all table typesDark header #2a2926 with white/45% label text. Row dividers #ede9e3. All numbers in Inconsolata. Column headers 10px uppercase. These never change regardless of table type.
◫
Alternating row colors — white / #faf9f7Odd rows white, even rows #faf9f7. Contrast is intentionally subtle — enough to track a row across wide columns without visual noise. Never use strong color for zebra striping in data tables.
↕
Hover and selected use indigo tint — text shifts accordinglyHover → #ddddf0 with dim text shifting to muted indigo. Selected/drawer open → #ddddf0 with text shifting to deep indigo #2a2860. Color alone never carries the state — font-weight: 500 is also applied on selected. The indigo family creates a clear "active zone" contrast against the warm neutral rows.
↔
Text left, numbers right — no exceptionsNames, descriptions, statuses → left-aligned, Manrope. Amounts, codes, IDs, dates → right-aligned, Inconsolata. Mixed columns (e.g. a ref code that's also clickable) follow the dominant content type — IDs are right, descriptions are left.
▷
Drawer appears as a side overlay — table stays intactClicking a row opens a floating panel anchored to the right edge of the table. Selected row highlights to #ddddf0 with text shifting to #2a2860. Closed with ✕ or by clicking another row. Only the "Open [record]" CTA navigates away from the table.
⚫
All GL numbers plain black — Dr, Cr, and BalanceDebit ≠ bad, credit ≠ good, balance has no polarity. All GL values are #1a1a1a. The column header labels carry structural distinction only. This matches Xero and QuickBooks — structure does the work, not color.
__
Clickable references use underline, not blueINV-, REC-, JE- references signal clickability through text-decoration underline with a muted underline color (#ccc). Blue is reserved for the number color system only.
02Modules
Modules
Recurring UI patterns — tax display, AI insights, and the actionable feed.
Module A — Tax display
Why tax withholding uses parentheses + red: The accounting standard globally — IFRS, Indonesian PSAK, IRS — uses parentheses (200.000,00) to indicate deductions, not a minus sign. Parentheses are unambiguous and print-safe. Red is applied only to the withheld amount (PPh 23) because it represents money that genuinely leaves the payer's hand — a real cash reduction, not just an accounting entry. PPN is a passthrough (collected then remitted) so it stays black. This pattern matches modern SaaS tools like Stripe and FreshBooks.

Parentheses for deductions, not minus signsThe accounting standard for negative/deduction amounts is (200.000,00) — not -200.000,00. Parentheses are internationally recognized (PSAK, IFRS, IRS), print-safe, and visually distinct from a zero. Stay consistent across all tax breakdowns.
🔴
Red only on the withheld (PPh) line — not on PPNPPh 23 withholding is money that leaves the payer's pocket immediately — a real cash deduction. Red here means "less money you receive," which is accurate. PPN is a passthrough collected on behalf of the government and remitted separately — it's not a loss, so it stays black.
⚫
Never use red for Dr/Cr values in the GLDebit is not inherently negative. Reserve red only for: (1) PPh withholding in tax breakdowns, (2) overdue amounts in bill lists, (3) negative deltas in dashboards. Anywhere else, red would mislead.
Module B — AI insight card

The insight card uses a dark background — distinct from all other surfaces. Body text stays in Manrope (not a serif), but the dark surface and eyebrow treatment create enough separation. AI voice leads with interpretation, not raw data.

✦ TradeOS AI · Insight
Revenue is up Rp 52 jt vs last month, mainly from PT Maju Jaya. But your AR is aging — 3 invoices totaling Rp 92 jt are overdue by more than 30 days.
⬛
Dark surface separates AI from UIThe #1a1a1a background is the only element in the product that uses this inversion. It creates an immediate visual signal: this is the system speaking, not a data field.
🎯
Lead with the takeaway, not the data"AR is aging" before the count. The interpretation comes first so users understand the implication before reading numbers.
✦
Amber for key figures inside the insightThe most critical number or phrase renders in amber (#c07a00) — same token used for warnings in the number color system, consistent across contexts.
